# TWINKOTALK_PRODUCT_EXPERIENCE_STRATEGY

> Status date: 2026-07-13  
> Current product stage: bounded prototype planning.

## 1. Product Strategy

TwinkoTalk is organized around one relationship: the user and Twinko. Features are not parallel apps; they are different ways Twinko accompanies the user.

### Product pillars

1. **Companion presence** — Twinko is visually and emotionally present.
2. **Reflection** — conversation and Tarot help users think through feelings and questions.
3. **Daily ritual** — Astrology gives a low-effort return moment.
4. **World transformation** — each mode has a distinct mood while remaining part of one universe.
5. **Healthy agency** — the user stays in control and is never guilted into returning.

## 2. Role of Each Current Mode

### Chat — relationship core

- Warm conversation surface.
- User expression.
- Foundation for future personalization and memory.
- Prototype uses scripts only.

### Tarot — reflective ritual

- Mystery, ceremony, and visual differentiation.
- Relationship / career / personal-growth questions.
- Potential future premium value, unvalidated.

### Astrology — daily ritual

- Fast, scannable daily content.
- Potential daily-return hypothesis.
- Sun-sign level in prototype.

### Meditation and Music — future universe

- Visible as future modes.
- Not interactive in the prototype.

## 3. Prototype Experience Principles

- Twinko before menus.
- One clear primary action per screen.
- Mode changes should feel like Twinko changes role or setting.
- Avoid long static information blocks where a character-led presentation works better.
- Avoid heavy animation that delays tasks.
- Disabled features must be understandable.
- Copy should be concise, warm, and natural in Traditional Chinese.

## 4. Prototype User Journey

1. Meet Twinko.
2. Create a lightweight profile.
3. Enter the shared companion world.
4. Choose Chat, Tarot, or Astrology.
5. Complete the mode experience.
6. Return to Home.

## 6. Feature-Zoo Prevention

Every new feature must answer:

- Why would Twinko offer this now?
- What user situation triggers it?
- How does it strengthen the companion relationship?
- Is it a mode, content, progression, or business layer?
- Can it be deferred without reducing the core promise?

Features that cannot answer these questions should not enter the roadmap.

## 7. Product Metrics by Stage

### Prototype

- Flow completion.
- Usability issues.
- Visual consistency.
- Perceived warmth.
- Perceived Tarot ritual quality.
- Astrology clarity.

### Beta / MVP later

- Activation.
- D7 / D30 retention.
- Meaningful sessions.
- User-reported helpfulness.
- Memory comfort.
- Safety incidents.
- WTP.

Do not use beta metrics as prototype acceptance criteria.
